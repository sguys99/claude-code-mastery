## 메타 프롬프트

```
당신은 솔로 개발자를 위한 PRD 전문가입니다.
아래 프로젝트 정보를 바탕으로 실용적이고 개발 가능한 PRD 문서를 작성해주세요.

## 프로젝트 개요

**프로젝트명**: Invoice Web Viewer (견적서 웹 뷰어)

**핵심 목표**:
노션에서 작성한 견적서를 클라이언트가 고유 URL을 통해 웹에서 확인하고 PDF로 다운로드할 수 있는 시스템

**사용자 시나리오**:
1. 관리자가 노션에서 견적서 데이터 입력
2. 시스템이 노션 데이터를 자동으로 동기화
3. 클라이언트에게 고유 견적서 URL 전달
4. 클라이언트가 웹에서 견적서 확인
5. 클라이언트가 PDF로 견적서 다운로드

## 기술 스택 (고정)

- **Framework**: Next.js 15.5.3 (App Router + Turbopack)
- **Runtime**: React 19.1.0 + TypeScript 5
- **Styling**: TailwindCSS v4 + shadcn/ui (new-york style)
- **Forms**: React Hook Form + Zod (필요시)
- **UI Components**: Radix UI + Lucide Icons
- **외부 연동**: Notion API

## PRD 작성 요구사항

### 1. 문서 구조
다음 섹션을 포함하여 작성:

1. **프로젝트 개요**
   - 프로젝트 이름 및 한줄 설명
   - 해결하려는 문제
   - 핵심 가치 제안

2. **사용자 정의**
   - 주요 사용자 (관리자/클라이언트)
   - 각 사용자별 니즈와 목표

3. **핵심 기능 (MVP 범위)**
   - 필수 기능 목록 (Must Have)
   - 우선순위 및 의존성
   - 각 기능별 상세 요구사항

4. **시스템 아키텍처**
   - 데이터 흐름도
   - 노션 연동 방식
   - 페이지 라우팅 구조

5. **데이터 모델**
   - 견적서 데이터 스키마
   - 노션 데이터베이스 필드 매핑

6. **UI/UX 요구사항**
   - 페이지별 와이어프레임 설명
   - 반응형 디자인 요구사항
   - PDF 출력 레이아웃

7. **기술적 고려사항**
   - 보안 (URL 토큰, 접근 제어)
   - 성능 (캐싱 전략)
   - SEO/메타데이터

8. **MVP 제외 사항**
   - 향후 버전에서 고려할 기능
   - 의도적으로 제외한 기능과 이유

9. **성공 지표**
   - MVP 완료 기준
   - 측정 가능한 목표

### 2. 작성 원칙

- **구체적으로**: 모호한 표현 대신 명확한 요구사항 기술
- **개발자 친화적**: 바로 구현 가능한 수준의 상세도
- **MVP 집중**: 최소 기능에 집중, 과도한 기능 배제
- **실용적**: 1인 개발자가 2-4주 내 구현 가능한 범위
